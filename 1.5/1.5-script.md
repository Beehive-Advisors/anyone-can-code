## PART 1 — READING CPU FACTS

### What is machine architecture?

**SPEAK**
> "A program does not run on a generic machine. It runs on a processor that understands a specific instruction set. Names like arm64 and x86_64 are telling us which family of machine instructions the processor can execute."

**SPEAK**
> "We also care about cores and caches. The processor is fast, memory is slower, and caches exist to keep useful data closer to the core."

### Run the demo

**TYPE**
```bash
echo "Architecture: $(uname -m)"
```

**OUTPUT**
```text
Architecture: arm64
```

**EXPLAIN**
> "This is the broad identity of the machine. The operating system is telling us that it is running on arm64."

**TYPE**
```bash
echo "CPU brand: $(sysctl -n machdep.cpu.brand_string)"
```

**OUTPUT**
```text
CPU brand: Apple M4 Pro
```

**EXPLAIN**
> "This is the human-readable processor model. It helps us orient ourselves, but it is still only the headline."

**TYPE**
```bash
sysctl hw.physicalcpu hw.logicalcpu hw.l1dcachesize hw.l1icachesize hw.l2cachesize
```

**OUTPUT**
```text
hw.physicalcpu: 14
hw.logicalcpu: 14
hw.l1dcachesize: 65536
hw.l1icachesize: 131072
hw.l2cachesize: 4194304
```

**EXPLAIN**
> "Now we can see structure, not just branding. The operating system is exposing the number of cores and the sizes of several cache levels, all in bytes."

### Exercise — Convert cache sizes

**SPEAK**
> "Take two of those byte values and convert them into units that are easier for humans to read."

**TYPE**
```bash
python3 -c "print('L1 data cache in KiB:', 65536 / 1024); print('L2 cache in MiB:', 4194304 / 1024 / 1024)"
```

**OUTPUT**
```text
L1 data cache in KiB: 64.0
L2 cache in MiB: 4.0
```

**EXPLAIN**
> "The hardware fact stayed the same. We only changed the units we used to describe it."

## PART 2 — READING MEMORY AS PAGES

### What is memory accounting?

**SPEAK**
> "Operating systems do not manage RAM as one giant lump. They manage it in fixed-size chunks called pages. That is why memory tools report categories like free, active, inactive, and wired."

**SPEAK**
> "A number like 24 gigabytes tells us how much memory is installed. A page report tells us how the kernel is accounting for that memory right now."

### Run the demo

**TYPE**
```bash
sysctl hw.memsize
```

**OUTPUT**
```text
hw.memsize: 25769803776
```

**EXPLAIN**
> "That is total installed memory in raw bytes. It is correct, but not very human-friendly."

**TYPE**
```bash
python3 - <<'PY'
import subprocess
mem = int(subprocess.check_output(['sysctl', '-n', 'hw.memsize'], text=True).strip())
print(f'Total memory in GiB: {mem / 1024 / 1024 / 1024:.2f}')
PY
```

**OUTPUT**
```text
Total memory in GiB: 24.00
```

**EXPLAIN**
> "Same underlying value, better unit."

**TYPE**
```bash
vm_stat | awk 'NR==1 || /Pages free|Pages active|Pages inactive|Pages wired down/'
```

**OUTPUT**
```text
Mach Virtual Memory Statistics: (page size of 16384 bytes)
Pages free:                               55102.
Pages active:                            655574.
Pages inactive:                          652725.
Pages wired down:                        144388.
```

**EXPLAIN**
> "This is closer to the kernel's own bookkeeping. The first line gives us the page size. The next lines tell us how many pages fall into several categories."

**TYPE**
```bash
python3 - <<'PY'
import subprocess, re
text = subprocess.check_output(['vm_stat'], text=True)
page_size = int(re.search(r'page size of (\d+) bytes', text).group(1))
free_pages = int(re.search(r'Pages free:\s+(\d+)\.', text).group(1))
print(f'Free pages: {free_pages}')
print(f'Page size: {page_size} bytes')
print(f'Free memory in MiB: {free_pages * page_size / 1024 / 1024:.2f}')
PY
```

**OUTPUT**
```text
Free pages: 53333
Page size: 16384 bytes
Free memory in MiB: 833.33
```

**EXPLAIN**
> "This is the key move. We start with page-level counters because that is how the kernel thinks, and then we convert them into a number a person can read quickly."

### Exercise — Compute wired memory

**SPEAK**
> "Now make the same calculation for wired memory instead of free memory."

**TYPE**
```bash
python3 - <<'PY'
import subprocess, re
text = subprocess.check_output(['vm_stat'], text=True)
page_size = int(re.search(r'page size of (\d+) bytes', text).group(1))
wired_pages = int(re.search(r'Pages wired down:\s+(\d+)\.', text).group(1))
print(f'Wired memory in MiB: {wired_pages * page_size / 1024 / 1024:.2f}')
PY
```

**OUTPUT**
```text
Wired memory in MiB: 2221.41
```

**EXPLAIN**
> "Same structure, different page counter."

## PART 3 — FROM DISKS TO BYTES

### What is a storage layout?

**SPEAK**
> "Files do not float in empty space. They sit inside a file system, on top of volumes, on top of containers or partitions, on top of a physical device."

**SPEAK**
> "After we look at that big-picture storage map, we are going to zoom all the way down and look at a file as raw bytes."

### Run the demo

**TYPE**
```bash
mkdir -p 1.5 && cd 1.5 && printf 'Hello\n' > hello.txt && ls -l hello.txt
```

**OUTPUT**
```text
-rw-r--r--@ 1 giacobbe  staff  6 Apr 20 16:11 hello.txt
```

**EXPLAIN**
> "This file is six bytes long. We have five letters and one newline byte."

**TYPE**
```bash
diskutil list | head -12
```

**OUTPUT**
```text
/dev/disk0 (internal, physical):
   #:                       TYPE NAME                    SIZE       IDENTIFIER
   0:      GUID_partition_scheme                        *500.3 GB   disk0
   1:             Apple_APFS_ISC Container disk2         524.3 MB   disk0s1
   2:                 Apple_APFS Container disk3         494.4 GB   disk0s2
   3:        Apple_APFS_Recovery Container disk1         5.4 GB     disk0s3

/dev/disk3 (synthesized):
   #:                       TYPE NAME                    SIZE       IDENTIFIER
   0:      APFS Container Scheme -                      +494.4 GB   disk3
                                 Physical Store disk0s2
   1:                APFS Volume Macintosh HD            12.6 GB    disk3s1
```

**EXPLAIN**
> "This is the storage hierarchy the operating system sees. It is showing the physical device and the higher-level APFS structures built on top of it."

**TYPE**
```bash
cd 1.5 && xxd -g 1 -l 16 hello.txt
```

**OUTPUT**
```text
00000000: 48 65 6c 6c 6f 0a                                Hello.
```

**EXPLAIN**
> "Now we can see the byte values directly. Forty-eight in hex is uppercase H. Zero-a is the newline at the end."

**TYPE**
```bash
cd 1.5 && xxd -b -l 6 hello.txt
```

**OUTPUT**
```text
00000000: 01001000 01100101 01101100 01101100 01101111 00001010  Hello.
```

**EXPLAIN**
> "Nothing about the file changed between the last two commands. Only the representation changed. First hex, then binary."

### Exercise — Change the file

**SPEAK**
> "Overwrite the file with different text, then inspect the bytes again."

**TYPE**
```bash
cd 1.5 && printf 'Hi\n' > hello.txt && xxd -g 1 -l 16 hello.txt
```

**OUTPUT**
```text
00000000: 48 69 0a                                         Hi.
```

**EXPLAIN**
> "The byte sequence changed because the file content changed. The operating system still stores text as bytes."

## PART 4 — CONVERTING BETWEEN NUMBER SYSTEMS AND TEXT

### What is representation?

**SPEAK**
> "A byte does not come with one built-in display format. Humans can show the same value as decimal, binary, hexadecimal, or text."

**SPEAK**
> "Python is a good tool here because the built-in functions are small and direct. They let us move between these representations without hiding the mechanics."

### Run the demo

**TYPE**
```bash
python3 -c "print('Decimal 72 -> binary:', bin(72)); print('Decimal 72 -> hex:', hex(72))"
```

**OUTPUT**
```text
Decimal 72 -> binary: 0b1001000
Decimal 72 -> hex: 0x48
```

**EXPLAIN**
> "This is one numeric value shown in two different notations. The prefixes tell us which base Python used."

**TYPE**
```bash
python3 -c "print('Character H -> code point:', ord('H')); print('Code point 72 -> character:', chr(72)); print('Binary 01001000 -> decimal:', int('01001000', 2)); print('Hex 48 -> decimal:', int('48', 16))"
```

**OUTPUT**
```text
Character H -> code point: 72
Code point 72 -> character: H
Binary 01001000 -> decimal: 72
Hex 48 -> decimal: 72
```

**EXPLAIN**
> "Now we can cross the line between text and numbers. Uppercase H maps to code point seventy-two, and the same value can be parsed back from binary or hex text."

**TYPE**
```bash
cd 1.5 && printf 'Hello\n' > hello.txt && python3 -c "import pathlib; data = pathlib.Path('hello.txt').read_bytes(); print('Byte values:', list(data))"
```

**OUTPUT**
```text
Byte values: [72, 101, 108, 108, 111, 10]
```

**EXPLAIN**
> "These decimal byte values line up with the hex and binary views from xxd. It is still the same file, just shown in another notation."

### Exercise — Convert e

**SPEAK**
> "Take the character e, turn it into its code point, then show that value in hex."

**TYPE**
```bash
python3 -c "value = ord('e'); print('Code point:', value); print('Hex:', hex(value))"
```

**OUTPUT**
```text
Code point: 101
Hex: 0x65
```

**EXPLAIN**
> "Once you have the numeric value, you can format it however you want."
