## PART 1 - Reading CPU Facts

### What is machine architecture?

**SPEAK**
> "A program does not run on a generic computer. It runs on a processor with a specific instruction set. When you see a name like arm64 or x86_64, that is the machine vocabulary the processor understands."

**SPEAK**
> "We also care about cores and caches. The CPU is very fast. Main memory is much slower. Caches exist so the processor can keep the most useful data close by."

### Run the demo

**TYPE**
```bash
echo "Architecture: $(uname -m)"
```

**OUTPUT**
```text
Architecture: arm64
```

**EXPLAIN**
> "This tells us the instruction-set family. On this machine, the operating system is running on arm64."

**TYPE**
```bash
echo "CPU brand: $(sysctl -n machdep.cpu.brand_string)"
```

**OUTPUT**
```text
CPU brand: Apple M4 Pro
```

**EXPLAIN**
> "This is the human-readable model string. It helps you identify the processor, but it is only the headline."

**TYPE**
```bash
sysctl hw.physicalcpu hw.logicalcpu hw.l1dcachesize hw.l1icachesize hw.l2cachesize
```

**OUTPUT**
```text
hw.physicalcpu: 14
hw.logicalcpu: 14
hw.l1dcachesize: 65536
hw.l1icachesize: 131072
hw.l2cachesize: 4194304
```

**EXPLAIN**
> "Now we are looking at structure, not branding. We can see core counts and cache sizes in bytes. That is the operating system exposing part of the processor's physical design."

### Exercise - Convert cache sizes

**SPEAK**
> "Take two of those byte values and convert them into units humans actually like reading."

**TYPE**
```bash
python3 -c "print('L1 data cache in KiB:', 65536 / 1024); print('L2 cache in MiB:', 4194304 / 1024 / 1024)"
```

**OUTPUT**
```text
L1 data cache in KiB: 64.0
L2 cache in MiB: 4.0
```

**EXPLAIN**
> "The underlying values did not change. We only changed the units we used to read them."

## PART 2 - Reading Memory As Pages

### What is memory accounting?

**SPEAK**
> "Operating systems do not manage RAM as one huge blob. They manage it in fixed-size chunks called pages. That is why memory tools often report categories like free, active, inactive, and wired."

**SPEAK**
> "A number like 24 gigabytes answers the question of how much memory is installed. A page report answers the question of how the kernel is using and tracking that memory right now."

### Run the demo

**TYPE**
```bash
sysctl hw.memsize
```

**OUTPUT**
```text
hw.memsize: 25769803776
```

**EXPLAIN**
> "That is total installed memory in raw bytes. It is accurate, but not very friendly for humans."

**TYPE**
```bash
python3 - <<'PY'
import subprocess
mem = int(subprocess.check_output(['sysctl', '-n', 'hw.memsize'], text=True).strip())
print(f'Total memory in GiB: {mem / 1024 / 1024 / 1024:.2f}')
PY
```

**OUTPUT**
```text
Total memory in GiB: 24.00
```

**EXPLAIN**
> "Same value, better unit. The machine has 24 GiB of memory."

**TYPE**
```bash
vm_stat | awk 'NR==1 || /Pages free|Pages active|Pages inactive|Pages wired down/'
```

**OUTPUT**
```text
Mach Virtual Memory Statistics: (page size of 16384 bytes)
Pages free:                               58672.
Pages active:                            650907.
Pages inactive:                          658467.
Pages wired down:                        142136.
```

**EXPLAIN**
> "The first line tells us the page size. The next lines tell us how many pages are in several categories. This is much closer to the kernel's actual bookkeeping."

**TYPE**
```bash
python3 - <<'PY'
import subprocess, re
text = subprocess.check_output(['vm_stat'], text=True)
page_size = int(re.search(r'page size of (\d+) bytes', text).group(1))
free_pages = int(re.search(r'Pages free:\s+(\d+)\.', text).group(1))
print(f'Free pages: {free_pages}')
print(f'Page size: {page_size} bytes')
print(f'Free memory in MiB: {free_pages * page_size / 1024 / 1024:.2f}')
PY
```

**OUTPUT**
```text
Free pages: 54185
Page size: 16384 bytes
Free memory in MiB: 846.64
```

**EXPLAIN**
> "This is the key move. We started with page counts, because that is how the kernel thinks. Then we converted the low-level counters into a human-sized number."

### Exercise - Compute wired memory

**SPEAK**
> "Now make the same calculation for wired memory instead of free memory."

**TYPE**
```bash
python3 - <<'PY'
import subprocess, re
text = subprocess.check_output(['vm_stat'], text=True)
page_size = int(re.search(r'page size of (\d+) bytes', text).group(1))
wired_pages = int(re.search(r'Pages wired down:\s+(\d+)\.', text).group(1))
print(f'Wired memory in MiB: {wired_pages * page_size / 1024 / 1024:.2f}')
PY
```

**OUTPUT**
```text
Wired memory in MiB: 2288.38
```

**EXPLAIN**
> "The pattern is the same. We only changed which counter we multiplied by the page size."

## PART 3 - From Disks To Bytes

### What is a disk layout?

**SPEAK**
> "Files do not float in empty space. They live inside a file system, on top of volumes, on top of containers or partitions, on top of a physical device."

**SPEAK**
> "After we look at that high-level storage map, we are going to zoom all the way in and look at a file as raw bytes."

### Run the demo

**TYPE**
```bash
cd 1.5 && printf 'Hello\n' > hello.txt && ls -l hello.txt
```

**OUTPUT**
```text
-rw-r--r--@ 1 giacobbe  staff  6 Apr 20 15:47 hello.txt
```

**EXPLAIN**
> "This file is six bytes long. We have five letters and one newline byte."

**TYPE**
```bash
diskutil list | head -12
```

**OUTPUT**
```text
/dev/disk0 (internal, physical):
   #:                       TYPE NAME                    SIZE       IDENTIFIER
   0:      GUID_partition_scheme                        *500.3 GB   disk0
   1:             Apple_APFS_ISC Container disk2         524.3 MB   disk0s1
   2:                 Apple_APFS Container disk3         494.4 GB   disk0s2
   3:        Apple_APFS_Recovery Container disk1         5.4 GB     disk0s3

/dev/disk3 (synthesized):
   #:                       TYPE NAME                    SIZE       IDENTIFIER
   0:      APFS Container Scheme -                      +494.4 GB   disk3
                                 Physical Store disk0s2
   1:                APFS Volume Macintosh HD            12.6 GB    disk3s1
```

**EXPLAIN**
> "This is the storage hierarchy the operating system sees. It shows the physical device and the APFS structures built on top of it."

**TYPE**
```bash
cd 1.5 && xxd -g 1 -l 16 hello.txt
```

**OUTPUT**
```text
00000000: 48 65 6c 6c 6f 0a                                Hello.
```

**EXPLAIN**
> "This is the same file, but now we can see the byte values directly. Forty-eight in hex is the byte for uppercase H. Zero-a is the newline."

**TYPE**
```bash
cd 1.5 && xxd -b -l 6 hello.txt
```

**OUTPUT**
```text
00000000: 01001000 01100101 01101100 01101100 01101111 00001010  Hello.
```

**EXPLAIN**
> "Nothing about the file changed between the last two commands. Only the representation changed. First hex. Now binary."

### Exercise - Change the file

**SPEAK**
> "Overwrite the file with different text, then inspect the bytes again."

**TYPE**
```bash
cd 1.5 && printf 'Hi\n' > hello.txt && xxd -g 1 -l 16 hello.txt
```

**OUTPUT**
```text
00000000: 48 69 0a                                         Hi.
```

**EXPLAIN**
> "The file is shorter now, and the byte sequence changed because the content changed. The operating system still stores text as bytes."

## PART 4 - Converting Between Number Systems And Text

### What is representation?

**SPEAK**
> "A byte does not come with meaning attached. Humans choose how to display it. We can show one value as decimal, hex, binary, or text."

**SPEAK**
> "Python is perfect for this because the built-in functions are small and direct. They let us switch views without bringing in any extra libraries."

### Run the demo

**TYPE**
```bash
python3 -c "print('Decimal 72 -> binary:', bin(72)); print('Decimal 72 -> hex:', hex(72))"
```

**OUTPUT**
```text
Decimal 72 -> binary: 0b1001000
Decimal 72 -> hex: 0x48
```

**EXPLAIN**
> "This is one numeric value shown in two different notations. The prefixes tell us which base Python used."

**TYPE**
```bash
python3 -c "print('Character H -> code point:', ord('H')); print('Code point 72 -> character:', chr(72)); print('Binary 01001000 -> decimal:', int('01001000', 2)); print('Hex 48 -> decimal:', int('48', 16))"
```

**OUTPUT**
```text
Character H -> code point: 72
Code point 72 -> character: H
Binary 01001000 -> decimal: 72
Hex 48 -> decimal: 72
```

**EXPLAIN**
> "Now we can cross the line between text and numbers. Uppercase H maps to code point seventy-two. The same value can be parsed from binary text or hex text and land back on the same integer."

**TYPE**
```bash
cd 1.5 && printf 'Hello\n' > hello.txt && python3 -c "import pathlib; data = pathlib.Path('hello.txt').read_bytes(); print('Byte values:', list(data))"
```

**OUTPUT**
```text
Byte values: [72, 101, 108, 108, 111, 10]
```

**EXPLAIN**
> "These decimal byte values line up with the hex and binary views from xxd. This is the same data again, just expressed differently."

### Exercise - Convert e

**SPEAK**
> "Take the character e, turn it into its code point, then show that value in hex."

**TYPE**
```bash
python3 -c "value = ord('e'); print('Code point:', value); print('Hex:', hex(value))"
```

**OUTPUT**
```text
Code point: 101
Hex: 0x65
```

**EXPLAIN**
> "Once you have the numeric value, you can format it however you want."
